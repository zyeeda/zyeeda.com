开发平台
Cdeio 是一款面向快速构建的企业级应用开发平台。在 Java 环境下，她融合了众多优秀的 组件与框架，以良好的设计、统一的封装、精简的 API 及完整定义的界面，让开发人员能够以意想不到的速度创建复杂的企业级应用程序

定义实体

使用 Cdeio 进行开发是从定义实体开始的。与大多数流行的 Java 框架类似，Cdeio 也是基于 Java Persistence API (JPA) 标准来实现数据可持久化的。

但与众不同的是，Cdeio 为开发者提供了强大而可扩展的自动生成能力，通过在实体类上添加 @Scaffold 标记，平台会根据一些配置为该实体生成诸如列表、新增、编辑、删除和查看等常用功能的前端界面和后端业务逻辑。



添加配置

Cdeio 的自动生成功能需要一些额外的配置，这些配置以类似 DSL 风格的语法来编写，主要用于描述实体类定义中缺失的内容。

scaffold.js 是配合自动生成功能使用的重要文件之一，这其中可以定义后端到前端的数据序列化规则，实体字段在页面上显示的名称，以及与表单有关的字段分组和布局信息等等。



That's all. Let's rock!

只需要这些，Cdeio 就可以为你生成全套的基本功能！

列表界面是操作实体数据的主要入口，默认由按钮栏、分页栏、表头和表体组成。可以通过在 scaffold.js 中添加其它配置，为列表界面定制新的功能。



新增界面

正如你所想象的，Cdeio 会为你准备好新增数据所需的界面。

在深入了解 Cdeio 的高级功能之后会发现，这样的表单可以实现得非常复杂：包括各种字段类型及验证、自由多变的布局风格，内联的数据列表以及相互嵌套的表单等等。一定可以满足你对表单界面的各种需求。



不只有界面

Cdeio 不单单能生成符合要求的表单，而且相关的前后端逻辑也是一并完整可用的。直接在界面上输入信息，点击确定，数据就会被写入数据库。



让我们继续摇滚

列表界面会在新增数据之后刷新。

选中一条数据，按钮栏会发生变化，多出来的按钮表示了针对该条数据可以进行的操作。这些功能操作都是真实有效地，而且也都是自动生成的，不需要任何多余的代码。

按钮栏上的按钮也可以根据需要来定制，无论是去掉默认的，增添新功能的，改变按钮的文字、图标、颜色、分组和排序等等都是可以实现的。甚至按钮在什么状态下会隐藏或显示也可以由你来随意控制。

三七原则
快速开发
Rapid Development

Rapid Development

快速开发是 Cdeio 一贯的追求。任何开发团队都想提升自身的效率，却无奈受困于很多重复或阻塞性质的工作上：比如从头搭建一个开发环境，实现各种繁琐而没有创造性的基础功能，以及等待没完没了的编译和重启过程等。Cdeio 的很多特性都是为了解决这些问题而诞生的
技术整合
Technology Integration

Technology Integration

技术整合要求统一基本的技术路线，和所有可能使用到的框架、组件和类库，不仅如此还要保证它们之间能够相互兼容。否则开发团队会因为分散的技术和不稳定的结合而无法实现有效的积累。Cdeio 平台以“独一无二”的理念，力求让团队找到技术整合的正确道路
资源同质
Resource Homogeneity

Resource Homogeneity

以 Cdeio 为平台，就相当于有了统一的技术整合基础，开发团队可以基于此定向培养和招募人才。更重要的是，资源的同质性让项目间调配变得容易：轻质资源可以被分配到任意的项目中，而优质资源又可以随时攻坚最棘手的技术问题。不论怎样，最终都可以提高资源的利用效率，从而节约对软件开发来说至关重要的人力资源成本
设计简化
Design Simplification

Design Simplification

在一个项目团队中，设计人员往往比开发人员更重要，因为他们设计成果的优劣，将直接决定后续开发能否成功。好的设计人员少之又少，而为一个复杂的系统进行完备的设计更是难上加难，那么至少可以让设计的过程变得简单而不那么痛苦。以 Cdeio 平台作为项目的基础，设计可以变得有据可循。还有现成的辅助工具可以将符合要求的设计文档进行导入，完成设计的同时，就完成了系统框架的搭建
外观统一
Appearance Unification

Appearance Unification

统一的外观之于产品，就好像统一的语言之于民族一样，可以在很短的时间内被贴上专属的标签。因此外观设计也通常被称作是一种语言，一种视觉效果的语言，用它来描述界面的布局、区域的配色、控件的样式以及对操作的反馈等等。使用 Cdeio 的目标之一就是为产品穿上统一风格的外装，在没有签名的情况下也能让用户时常联想起作者
质量保证
Quality Assurance

Quality Assurance

Cdeio 的自动生成能力，能够为所开发的系统提供最基本的质量保证。从界面到数据再到业务逻辑，每一个质量环节都由平台严格把控，从而减少开发人员的低级失误。不会再有不一致的界面、不经验证的表单、错误的数据访问以及不合章法的代码片段出现在经由平台生成的产品中

服务端 JavaScript

Server-side JavaScript

服务器端 JavaScript 是 Cdeio 平台的最大特色之一，也是颠覆传统 Spring Struts 和 Hibernate（SSH）开发模式的主要所在。作为脚本编程语言，JavaScript 相比 Java 而言具有天然的快速开发优势，服务器端 JavaScript 可以做到代码修改后无需编译与重启，而且还能够使得前后端开发技术得到统一：同一种编程思想，同一套组件类库，一次学习可以前后端通用。

微内核框架

Micro Kernel Framework

Cdeio 的后端是基于 Spring 框架封装的 Java 运行时环境，整合了各种核心业务组件，并在此之上添加了 JavaScript 脚本执行引擎，还基于该引擎构建了全套的 MVC 开发模型。除了核心代码和必要的封装代码使用 Java 语言编写之外，平台的大部分功能都是使用 JavaScript 编写而成的。

面向实体编程

Entity Oriented Programming

企业级应用系统都是以数据为核心的，数据在系统内部体现为实体以及实体跟实体之间的关系。无论是数据的增删改查、流程变迁、规则变换以及统计分析等，都跟实体脱离不了干系。因此 Cdeio 突出面向实体编程的理念，围绕实体扩充各种与业务相关的功能与界面。

动态脚手架

Dynamic Scaffold

系统开发过程中，有大量工作是繁琐且重复的，开发人员纠结于各种实现细节而无法集中精力解决真正的业务问题。Cdeio 平台的动态脚手架就是以自动生成的方式辅助实现一些需要大量工作才能完成的通用功能：比如带有过滤条件的列表界面、树和列表联动的界面、各种业务表单的查看与提交，根据流程图自动生成界面和调用逻辑等。

约定优于配置

Convention over Configuration

约定优于配置的特性，是将一些约定俗成或规范化的东西，直接印刻在系统的实现过程中，而不是通过引入大量配置来支撑系统的定制特性。只要按照约定方式布局和命名文件，Cdeio 就能够自动加载和运行，只在必须存在配置的情况下才引入少量的配置文件，以支持系统的灵活性和可变性。

松耦合的前后端绑定

Loosely Frontend & Backend Binding

Web 应用程序一般分为前端和后端系统，而通常情况下两者都是紧密结合的，前端界面都是由后端系统的视图层负责输出的。Cdeio 采用前后端程序独立的松耦合架构，前端应用和后端应用可以独立开发，并使用事先约定好的协议进行通讯。这样就使得两部分应用都可以独立存在，并支持适配其他的异构系统，如移动终端或者嵌入式设备等。

丰富的前后端组件库

Plenty of Frontend & Backend Component Libraries

Cdeio 整合并封装了众多流行且前沿的前后端框架及组件，免去了开发人员自行选择所带来的进度延迟和技术风险。平台级别的组件库，经过缜密的选型、细致的设计、统一的封装，配合动态脚手架功能，能够使用优雅的配置参数或 API 调用，驱动各种功能强大的第三方组件，以满足复杂的企业级应用系统的需要。

扁平化的用户界⾯设计

Flat-styled User Interface Design

扁平化是如今界面设计的潮流与趋势。Cdeio 使用了一套非常精彩的基于 Bootstrap 的扁平化主题，界面元素丰富，样式统一，颜色清新而不单调，丰富却不烦乱，还能够将各种第三方组件的样式整合起来。再配合 HTML 5、CSS 3 和 Ajax 等技术，进而实现丰富而生动的界面效果。

开箱即用

Out of Box Is Ready

功能强大的平台往往难于上手，一个重要的原因在于系统的初始化过程不够简单。Cdeio 随行提供的安装程序和 Starter Kit 新手包，可以简化初识人员的学习过程。在开箱即用的基础上，逐渐自行修改配置和添加功能，进而做到独立使用。另外 Cdeio 还提供了各种类型的参考文档，包括“开发人员手册”、“API 手册”、“示例代码与说明”和“安装部署手册”等，可以指引开发人员从零开始深入学习平台的绝大部分功能。
