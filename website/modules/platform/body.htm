定义实体

使用 Zyeeda Cdeio 进行开发是从定义实体开始的。与大多数流行的 Java 框架类似，Zyeeda Cdeio 也是基于 Java Persistence API (JPA) 标准来实现数据可持久化的。

但与众不同的是，Zyeeda Cdeio 为开发者提供了强大而可扩展的自动生成能力，通过在实体类上添加 @Scaffold 标记，平台会根据一些配置为该实体生成诸如列表、新增、编辑、删除和查看等常用功能的前端界面和后端业务逻辑。



添加配置

Zyeeda Cdeio 的自动生成功能需要一些额外的配置，这些配置以类似 DSL 风格的语法来编写，主要用于描述实体类定义中缺失的内容。

scaffold.js 是配合自动生成功能使用的重要文件之一，这其中可以定义后端到前端的数据序列化规则，实体字段在页面上显示的名称，以及与表单有关的字段分组和布局信息等等。



That's all. Let's rock!

只需要这些，Zyeeda Cdeio 就可以为你生成全套的基本功能！

列表界面是操作实体数据的主要入口，默认由按钮栏、分页栏、表头和表体组成。可以通过在 scaffold.js 中添加其它配置，为列表界面定制新的功能。



新增界面

正如你所想象的，Zyeeda Cdeio 会为你准备好新增数据所需的界面。

在深入了解 Zyeeda Cdeio 的高级功能之后会发现，这样的表单可以实现得非常复杂：包括各种字段类型及验证、自由多变的布局风格，内联的数据列表以及相互嵌套的表单等等。一定可以满足你对表单界面的各种需求。



不只有界面

Zyeeda Cdeio 不单单能生成符合要求的表单，而且相关的前后端逻辑也是一并完整可用的。直接在界面上输入信息，点击确定，数据就会被写入数据库。



让我们继续摇滚

列表界面会在新增数据之后刷新。

选中一条数据，按钮栏会发生变化，多出来的按钮表示了针对该条数据可以进行的操作。这些功能操作都是真实有效地，而且也都是自动生成的，不需要任何多余的代码。

按钮栏上的按钮也可以根据需要来定制，无论是去掉默认的，增添新功能的，改变按钮的文字、图标、颜色、分组和排序等等都是可以实现的。甚至按钮在什么状态下会隐藏或显示也可以由你来随意控制。

快速开发
Rapid Development

Rapid Development

快速开发是 Zyeeda Cdeio 一贯的追求。任何开发团队都想提升自身的效率，却无奈受困于很多重复或等待性质的工作上：比如从头搭建一个开发环境，实现各种繁琐而没有创造性的基础功能，还有等待没完没了的编译和重启过程等。
技术整合
Technology Integration

Technology Integration

技术整合要求统一基本的技术路线，和所有可能使用到的框架、组件和类库，不仅如此还要保证它们之间能够相互兼容。否则开发团队会因为分散的技术和不稳定的结合而无法实现有效的积累。Zyeeda Cdeio 平台以“独一无二”的理念，力求让团队找到技术整合的正确道路。
资源同质
Resource Homogeneity

Resource Homogeneity

以 Zyeeda Cdeio 为平台，就相当于有了统一的技术整合基础，开发团队可以基于此定向培养和招募人才。更重要的是，资源的同质性让项目间调配变得容易：轻质资源可以被分配到任意的项目中，而优质资源又可以随时攻坚最棘手的技术问题。不论怎样，最终都可以提高资源的利用效率，从而节约对软件开发来说至关重要的人力资源成本。
设计简化
Design Simplification

Design Simplification

在一个项目团队中，设计人员往往比开发人员更重要，因为他们设计成果的优劣，将直接决定后续开发能否成功。好的设计人员少之又少，而为一个复杂的系统进行完备的设计更是难上加难，那么至少可以让设计的过程变得简单而不那么痛苦。以 Zyeeda Cdeio 平台作为项目的基础，设计可以变得有据可循。还有现成的辅助工具可以将符合要求的设计文档进行导入，完成设计的同时，就完成了系统框架的搭建。
外观统一
Appearance Unification

Appearance Unification

统一的外观之于产品，就好像统一的语言之于民族一样，可以在很短的时间内被贴上专属的标签。因此外观设计也通常被称作是一种语言，一种视觉效果的语言，用它来描述界面的布局、区域的配色、控件的样式以及对操作的反馈等等。使用 Zyeeda Cdeio 的目标之一就是为产品穿上统一风格的外装，在没有签名的情况下也能让用户时常联想起作者。
